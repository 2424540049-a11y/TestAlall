# 沪铝小时级历史数据源调研

生成日期：2026-06-23

## 目标

寻找能覆盖 2005 年至今的沪铝小时级历史数据，并最终导入 app 的 `DATA` 目录，让 `1h/3h/5h` K 线不再只依赖新浪近期分钟线。

## 本地已验证

当前 app 的小时线来自新浪分钟线接口。实测：

```text
接口：http://localhost:8787/api/kline?product=al&symbol=nf_AL0&interval=1h&start=2005-01-01&end=2026-06-23&limit=30000
返回：1023 根
范围：2025-12-05 00:00:00 至 2026-06-23 15:00:00
```

AkShare 的 `futures_zh_minute_sina(symbol="AL0", period="60")` 底层同样是新浪，结果一致。

东方财富免费历史 K 线接口实测：

```text
日线 klt=101：2005-01-03 至 2026-06-23，可用
60分钟 klt=60：仅 2026-05-29 至 2026-06-23，不够
5分钟 klt=5：仅近期，不够
```

## 结论

没有找到免费、无需登录、可直接覆盖 2005 年至今沪铝小时线的公开接口。

最接近目标的数据源是聚宽 JQData 和 BigQuant。

聚宽 JQData 文档显示商品期货 tick 数据、指定时间周期的分钟/日行情数据、`get_bars()` 均覆盖 2005 年至今，并且支持期货连续合约。这条路线可能最适合直接获取沪铝主力连续小时线。

BigQuant 的期货分钟行情表 `cn_future_bar1m / cn_future_bar1m_c` 标明数据起始时间为 2005-01-01，字段包含开高低收、成交量、成交额、持仓量，并且有期货主力合约映射表。它可以用来构建沪铝主力连续分钟线，再聚合成 1h/3h/5h。

## 数据源对比

| 数据源 | 能否覆盖 2005 至今小时级 | 是否需要账号/付费 | 备注 |
|---|---:|---:|---|
| 新浪 / AkShare | 否 | 否 | 沪铝 60 分钟实测最早 2025-12-05 |
| 东方财富免费接口 | 否 | 否 | 日线可到 2005；分钟/小时只给近期 |
| 上海期货交易所公开下载 | 否 | 否 | 公开页面主要是日周/月年/统计下载，不是完整历史分钟线 |
| 天勤 TqSdk | 否 | 需账号，免费/付费视权限 | 文档说明每个 K 线序列最多最后 8000 根，无法覆盖 2005 至今 |
| 东财掘金量化 | 部分 | 需账号/权限 | 文档说明 SHFE 历史 Bar 的分钟级范围为 2017-01-01 至今 |
| Tushare Pro | 部分 | 期货历史分钟单独权限 | 权限表写期货历史分钟从 2010 年起，1/5/15/30/60 分钟 |
| 聚宽 JQData | 是，最匹配 | 需账号/SDK权限 | 文档写商品期货 tick、分钟/日行情、`get_bars()` 为 2005 年至今，并支持连续主力 |
| RQData / 米筐 | 可能 | 需购买或试用 | 支持期货分钟/tick 和主力连续合约，需账号 token |
| BigQuant | 是，最匹配 | 需账号/数据权限 | `cn_future_bar1m` 起始 2005-01-01；可配合 `cn_future_dominant` 拼主力 |
| Barchart Premier | 可能 | 付费 | 有 SHFE Aluminum intraday 下载，但不一定适配国内主连口径 |

## 推荐路线

优先路线：

```text
聚宽 JQData
-> 安装 jqdatasdk 并登录账号
-> 用 AL8888.XSGE 或 AL9999.XSGE 连续合约口径获取 60m 数据
-> 如需 3h/5h，再用 60m 聚合
-> 保存到 DATA/aluminum_AL0_1h_2005_20260622.json
-> 修改 server.js，小时线优先读取本地 DATA，2026-06-22 之后再接在线增量
```

并行可选路线：

```text
BigQuant
-> 下载 cn_future_dominant 中 AL 的每日主力映射
-> 下载 cn_future_bar1m 中对应真实合约的 1 分钟数据
-> 按主力映射拼成沪铝主连
-> 聚合为 1h / 3h / 5h
-> 保存到 DATA/aluminum_AL0_1h_2005_20260622.json
-> 修改 server.js，小时线优先读取本地 DATA，2026-06-22 之后再接在线增量
```

备选路线：

```text
Tushare Pro
-> 可做到约 2010 至今
-> 如果接受少 2005-2009，这条路线文档更清晰
```

```text
东财掘金量化
-> 可做到约 2017 至今
-> 比新浪长很多，但仍不满足 2005 起
```

## app 需要的数据格式

建议导入文件：

```text
DATA/aluminum_AL0_1h_2005_20260622.json
DATA/aluminum_AL0_3h_2005_20260622.json
DATA/aluminum_AL0_5h_2005_20260622.json
```

建议 JSON 结构：

```json
{
  "symbol": "AL0",
  "interval": "1h",
  "source": "BigQuant cn_future_bar1m + cn_future_dominant",
  "start": "2005-01-04 09:00:00",
  "end": "2026-06-22 15:00:00",
  "candles": [
    {
      "date": "2005-01-04 10:00:00",
      "open": 0,
      "high": 0,
      "low": 0,
      "close": 0,
      "volume": 0,
      "amount": 0,
      "openInterest": 0,
      "sourceContract": "AL0503"
    }
  ]
}
```

## 下一步

需要用户提供以下任一项：

1. 聚宽 JQData 账号，或从聚宽导出的 AL 主力连续 60m CSV。
2. BigQuant 账号/数据权限，或从 BigQuant 导出的 AL 分钟数据 CSV。
3. RQData/米筐账号 token，或导出的 AL 分钟数据 CSV。
4. Tushare Pro 分钟权限 token，接受数据从 2010 年开始。
5. 任意供应商导出的沪铝主连 1h CSV。

拿到数据后，就可以把小时级本地 DATA 接入 app。
